# Joinzer — User Research & Interview Notes

_Last updated: July 17, 2026_

> ⚠️ **This doc is empty of real data — and that is the single most important fact about Joinzer right now.** No organizer or player has used the product; there are zero interviews. This is a **container**: a template to capture input the moment it arrives, plus a synthesis section that accumulates patterns as interviews add up. Filling this doc is what upgrades the rest of `docs/strategy/` from hypothesis to fact.
>
> When you (Claude) reason from this doc, remember: **absence of evidence here means the personas/market/pricing docs are still guesses.** Prioritize helping get the first interviews done over refining untested theory.

## How to use this doc

1. Before an organizer conversation, pull the validation questions from `customers-and-personas.md` (§ "Questions to validate") and `open-decisions.md`.
2. During/after, fill one **Interview entry** below (copy the template).
3. After every few interviews, update **Synthesis** with what's repeating.
4. When something contradicts a strategy doc, update *that* doc and note it here — the codebase-wins discipline, applied to strategy: **real user input wins over assumption.**
